# MCC Keyword Research Tool — Design Specification

## Overview

Build a command-line Python tool that uses the Google Ads API `KeywordPlanIdeaService` to generate keyword ideas at scale across an MCC (My Client Center) with ~40 Google Ads client accounts. The tool pulls each account's top-performing keywords, feeds them as seeds into the Keyword Planner API, and outputs a structured spreadsheet with keyword opportunities per account — volume, competition, CPC estimates, and relevance scoring for the pool/spa industry vertical.

This tool fills a gap that Google Ads Scripts cannot: programmatic access to Keyword Planner data. It complements an existing suite of MCC audit scripts that handle search term analysis, negative keywords, QS optimization, and ad copy builds. The search term report shows what's already converting; this tool discovers what people are searching that the accounts haven't captured yet.

## Architecture

```
CLI Tool (Python 3.10+)
├── auth.py           — Google Ads API OAuth2 + developer token auth
├── config.yaml       — MCC ID, credentials, output settings
├── seed_extractor.py — Pulls top keywords per account via Google Ads API reporting
├── keyword_planner.py — Calls KeywordPlanIdeaService with seeds
├── scorer.py         — Scores/filters results for pool/spa relevance
├── writer.py         — Writes output to Google Sheets or Excel
└── main.py           — CLI entrypoint, orchestrates the pipeline
```

- **Python 3.10+**

## Authentication & Setup

### Prerequisites the user needs before running this tool:

1. **Google Ads Developer Token** — Applied for at https://developers.google.com/google-ads/api/docs/get-started/dev-token. SSP's MCC qualifies. The token is tied to the MCC manager account. Start with Test Account access for development, then apply for Basic or Standard access for production.

2. **OAuth2 Client Credentials** — Created in Google Cloud Console:
   - Create a project (or use existing)
   - Enable the Google Ads API
   - Create OAuth2 Client ID (Desktop application type)
   - Download `client_secret.json`

3. **Refresh Token** — Generated by running the OAuth2 flow once:
   - The tool should include a `setup` subcommand that runs the OAuth2 consent flow
   - User authenticates in browser, grants access to Google Ads
   - Tool stores the refresh token in `config.yaml` or a separate `.token` file
   - Subsequent runs use the refresh token (no browser needed)

4. **MCC Account ID** — The top-level manager account ID (XXX-XXX-XXXX format)

### config.yaml structure:

```yaml
google_ads:
  developer_token: "YOUR_DEVELOPER_TOKEN"
  client_id: "YOUR_CLIENT_ID.apps.googleusercontent.com"
  client_secret: "YOUR_CLIENT_SECRET"
  refresh_token: "YOUR_REFRESH_TOKEN"
  login_customer_id: "1234567890"  # MCC account ID, no dashes

output:
  type: "google_sheets"  # or "excel"
  spreadsheet_url: "https://docs.google.com/spreadsheets/d/..."  # for google_sheets
  excel_path: "./keyword_research_output.xlsx"  # for excel

settings:
  lookback_days: 90
  min_seed_conversions: 2      # Only use keywords with 2+ conversions as seeds
  min_seed_clicks: 10          # Or 10+ clicks as alternate seed qualification
  max_seeds_per_account: 15    # Cap seeds to avoid API quota issues
  max_ideas_per_seed: 50       # Ideas returned per seed group
  min_monthly_searches: 50     # Filter out very low volume ideas
  language_id: "1000"          # English
  # Country code is pulled per-account from geo targeting; fallback:
  default_country_code: "2840"  # US
```

## Pipeline: Step by Step

### Step 1: seed_extractor.py — Pull Top Seeds Per Account

For each account in the MCC:

1. Use `GoogleAdsService.SearchStream` to query the `keyword_view` resource:

```sql
SELECT
  campaign.name,
  ad_group.name,
  ad_group_criterion.keyword.text,
  ad_group_criterion.keyword.match_type,
  ad_group_criterion.quality_info.quality_score,
  metrics.conversions,
  metrics.clicks,
  metrics.impressions,
  metrics.cost_micros,
  metrics.conversions_from_interactions_rate
FROM keyword_view
WHERE ad_group_criterion.status = 'ENABLED'
  AND campaign.status = 'ENABLED'
  AND metrics.impressions > 0
  AND segments.date DURING LAST_90_DAYS
ORDER BY metrics.conversions DESC, metrics.clicks DESC
LIMIT 200
```

2. Score each keyword for seed quality:
   - Keywords with `conversions >= min_seed_conversions` → top tier seeds
   - Keywords with `clicks >= min_seed_clicks` and `quality_score >= 6` → secondary seeds
   - Deduplicate by keyword text (same keyword in multiple ad groups → keep highest-converting version)
   - Cap at `max_seeds_per_account`

3. Also pull the account's geo targeting to determine the location IDs for the Keyword Planner request:

```sql
SELECT
  campaign.name,
  campaign_criterion.location.geo_target_constant
FROM campaign_criterion
WHERE campaign_criterion.type = 'LOCATION'
  AND campaign.status = 'ENABLED'
```

Map `geo_target_constant` resource names to location IDs. If no geo targeting is found, fall back to `default_country_code`.

4. Output per account: `{ account_name, account_id, seeds: [...], location_ids: [...] }`

### Step 2: keyword_planner.py — Generate Keyword Ideas

For each account's seed set:

1. Call `KeywordPlanIdeaService.GenerateKeywordIdeas`:

```python
request = client.get_type("GenerateKeywordIdeasRequest")
request.customer_id = account_id
request.language = f"languageConstants/{language_id}"
request.geo_target_constants = [f"geoTargetConstants/{loc}" for loc in location_ids]
request.keyword_plan_network = client.enums.KeywordPlanNetworkEnum.GOOGLE_SEARCH
request.keyword_seed.keywords.extend(seed_keywords)
```

2. Parse the response. Each result contains:
   - `text` — the keyword idea
   - `keyword_idea_metrics.avg_monthly_searches` — average monthly volume
   - `keyword_idea_metrics.competition` — LOW / MEDIUM / HIGH enum
   - `keyword_idea_metrics.competition_index` — 0-100 numeric competition
   - `keyword_idea_metrics.low_top_of_page_bid_micros` — estimated low CPC
   - `keyword_idea_metrics.high_top_of_page_bid_micros` — estimated high CPC
   - `keyword_idea_metrics.monthly_search_volumes` — 12-month trend array

3. Filter results:
   - Remove ideas with `avg_monthly_searches < min_monthly_searches`
   - Remove ideas that already exist as active keywords in this account (cross-reference against the keyword list from Step 1)
   - Remove ideas that match the account's existing negative keyword list

4. Store raw results per account for scoring in Step 3.

### Rate Limiting & Quotas

- The Keyword Planner API has daily limits (varies by developer token access level)
- Basic access: 15,000 operations per day
- Batch seeds into groups of 10-15 per request to minimize API calls
- Add 1-second delay between requests per account
- Add 2-second delay between accounts
- Log all API calls and remaining quota if available
- If rate limited, save progress and allow resuming

### Step 3: scorer.py — Score & Rank for Pool/Spa Relevance

Each keyword idea gets a composite score (0-100) based on:

**Volume Score (0-25):**
- 500+ monthly searches → 25
- 200-499 → 20
- 100-199 → 15
- 50-99 → 10

**Competition Score (0-25):**
- Low competition → 25 (less competitive = easier to rank)
- Medium → 15
- High → 10 (expensive but validates demand)

**CPC Efficiency Score (0-25):**
- Compare estimated CPC to account's current average CPC
- If idea CPC < account avg CPC → 25 (cheaper traffic opportunity)
- If idea CPC ≈ account avg CPC (within 20%) → 15
- If idea CPC > account avg CPC by 50%+ → 5

**Industry Relevance Score (0-25):**

Apply a relevance classifier using keyword matching against pool/spa industry terms:

```python
HIGH_RELEVANCE = [
    "pool builder", "pool construction", "pool contractor", "pool company",
    "pool installation", "pool remodel", "pool renovation", "pool repair",
    "pool resurfacing", "pool plastering", "pool service", "pool cleaning",
    "pool maintenance", "inground pool", "gunite pool", "fiberglass pool",
    "vinyl pool", "custom pool", "pool cost", "pool price", "pool financing",
    "hot tub", "spa", "swim spa", "jacuzzi", "hot tub dealer",
    "hot tub sale", "hot tub near me", "outdoor living", "patio",
    "pool deck", "pool coping", "pool tile", "pool plaster",
    "pool enclosure", "pool cover", "pool heater", "pool pump",
    "pool filter", "pool light", "salt water pool", "pool design",
    "backyard pool", "swimming pool", "plunge pool", "cocktail pool"
]

MEDIUM_RELEVANCE = [
    "backyard", "outdoor", "landscape", "hardscape", "paver",
    "concrete", "deck", "pergola", "gazebo", "fence",
    "water feature", "fountain", "fire pit", "outdoor kitchen",
    "screen enclosure", "lanai"
]

LOW_RELEVANCE_NEGATIVE = [
    "pool table", "billiard", "car pool", "carpool", "gene pool",
    "pool party", "pool game", "pool noodle", "pool float", "pool toy",
    "inflatable", "intex", "bestway", "above ground",
    "walmart", "amazon", "home depot", "lowes",
    "diy", "how to", "youtube", "tutorial",
    "jobs", "hiring", "career", "salary",
    "used", "craigslist", "free"
]
```

- Contains HIGH_RELEVANCE term → 25
- Contains MEDIUM_RELEVANCE term → 15
- Contains no relevance signals → 8
- Contains LOW_RELEVANCE_NEGATIVE term → 0 (flag as negative keyword candidate)

**Priority classification:**
- Score 75-100 → HIGH — add to account immediately
- Score 50-74 → MEDIUM — review and likely add
- Score 25-49 → LOW — add if budget allows
- Score 0-24 → SKIP or NEGATIVE — irrelevant or too expensive

### Step 4: writer.py — Output to Spreadsheet

Generate one sheet tab per account (matching the pattern of the existing MCC Ad Build Export script).

**Tab: _INDEX**

Summary across all accounts:
- Total keyword ideas generated
- Ideas per priority tier (HIGH / MEDIUM / LOW)
- Accounts with most opportunities
- Runtime and settings used

**Tab: [Account Name] (one per account)**

Section 1: SEED KEYWORDS USED
| Keyword | Match Type | Conversions | Clicks | CPA | QS |
Show what seeds were fed to the Keyword Planner so the contractor understands the basis.

Section 2: HIGH PRIORITY KEYWORD IDEAS (Score 75+)
| Keyword Idea | Monthly Vol | Competition | Est. Low CPC | Est. High CPC | Score | Relevance | Already Negative? | Suggested Match Type | Suggested Ad Group |

- **Suggested Match Type:** Exact for 1-2 word ideas, Phrase for 3+ words
- **Suggested Ad Group:** Match to existing ad group by theme similarity (compare idea text against existing ad group keyword themes; suggest the closest match, or "NEW AD GROUP" if no existing group fits)

Section 3: MEDIUM PRIORITY KEYWORD IDEAS (Score 50-74)
Same columns as HIGH.

Section 4: NEGATIVE KEYWORD CANDIDATES
Ideas flagged as irrelevant (LOW_RELEVANCE_NEGATIVE matches). These should be added as campaign-level negatives to prevent future waste.

Section 5: 12-MONTH TREND DATA
For top 20 ideas by score, show the monthly search volume trend (12 values). This reveals seasonality — pool keywords spike March-June, hot tub keywords spike October-December. Useful for bid scheduling.

## CLI Interface

```bash
# First-time setup: run OAuth2 flow
python main.py setup

# Full run: all accounts
python main.py run

# Single account
python main.py run --account "Magnolia Custom Pools"

# Dry run: pull seeds only, don't call Keyword Planner
python main.py run --dry-run

# Resume after rate limit or timeout
python main.py run --resume

# Output to local Excel instead of Google Sheets
python main.py run --output excel

# Custom lookback and thresholds
python main.py run --lookback 180 --min-conversions 1 --max-seeds 20
```

## Error Handling

- **Auth failure:** Clear message with link to setup instructions
- **Rate limit (429):** Save progress to `.progress.json`, log which accounts are done, print resume command
- **Account access error:** Skip account, log error, continue to next
- **No seeds found:** Log warning ("Account X has no qualifying keywords — check if campaigns are active"), skip to next
- **API quota exhausted:** Save progress, print daily quota reset time (midnight Pacific)

## Key Constraints & Notes

1. **Google Ads API version:** Use the latest stable version (v18 as of early 2026). The google-ads Python library handles version management.

2. **MCC-level access:** All API calls use `login_customer_id` set to the MCC ID. The tool iterates child accounts using `CustomerService.ListAccessibleCustomers` or `ManagedCustomerService`.

3. **Keyword Planner requires active spend:** Google returns limited or no data for accounts with $0 recent spend. The tool should detect this and log a warning rather than failing.

4. **Volume data is ranges, not exact:** Unless the account is actively spending on Google Ads (which ours are), Keyword Planner returns volume as ranges (e.g., 100-1K). The tool should use the midpoint for scoring and note this in the output.

5. **Location targeting matters:** A keyword planner query for "pool builder" targeted at Houston TX returns different volume and CPC than one targeted nationally. The tool pulls each account's actual geo targeting and uses it — this gives locally accurate data.

6. **The tool is read-only:** It queries data and generates reports. It never modifies account settings, keywords, or ads. All changes are made manually or by the contractor using the output.

7. **Complement to search term report:** The existing MCC Ad Build Export script surfaces converting search terms (what's already working). This tool surfaces what people are searching that the account isn't capturing yet. The two outputs together give the contractor a complete keyword expansion playbook: proven converters + untapped demand.

## Testing Strategy

- **test_scorer.py:** Unit tests for relevance scoring. Feed known pool/spa keywords and verify HIGH scores. Feed known irrelevant keywords (pool table, carpool) and verify LOW/SKIP scores.
- **test_seed_extractor.py:** Mock GAQL responses, verify seed selection logic (conversion threshold, dedup, cap).
- **Manual integration test:** Run against 1-2 accounts with `--dry-run` to verify auth and seed extraction. Then full run on a single account to verify Keyword Planner output.
